Deploying to Neocities from GitHub Guide

C/U/ID: 2023-05-05/2023-05-05/G003

1. Why would I want to deploy to Neocities from GitHub?

Deploying to Neocities from GitHub allows you to take advantage of git and GitHub's features while keeping your site hosted on Neocities. Doing this also makes it very easy to create a mirror of your Neocities page on GitHub Pages (such as I did). It also means that you can use an external editor, such as Visual Studio Code or VSCodium, instead of the clunky Neocities web editor.

2. Setting up a GitHub repository

This guide will assume that you already know how to use git and GitHub.

First, make a new repository on GitHub. If you plan on mirroring your site on GitHub Pages, name it [GitHub username].github.io (e.g. mine is liassica.github.io) and make it public, otherwise call it whatever you like (such as 'neocities-site', etc.). Once you have it created, clone it to a local repository.

Next, download your Neocities site. This can be done from the Neocities dashboard by clicking 'Download entire site' below your files. Unzip the archive into a folder in your repository. If you plan to mirror on GitHub pages, call this folder docs, otherwise name it whatever you like (e.g. site, public, etc.). The reason why we do this is because Neocities has a limited number of allowed filetypes, and deploying to Neocities from the root folder of the repository can break things.

After that, create a folder in the root of your repository called .github. In that folder, create another folder called workflows. Now you should have in your root folder something like /.github/workflows and /docs. Next is the actual workflow file. Download this file from my repository and place it in your workflows folder. If you named your site folder something other than docs, open this file and change the line dist_dir to match the name of your folder (e.g. dist_dir: site, etc.).

Next, we need to add a Neocities API key as a secret to the repository. Navigate to https://neocities.org/settings/[neocities username]#api_key. This will show your API key. On your GitHub repository, go to Settings > Secrets and variables > Actions > New repository secret. Name the secret NEOCITIES_API_KEY and paste the key you got from Neocities as the value. Be very careful with this API key, as it allows anyone or anything with access to it to change anything on your site.

Now all that's left is to push to your GitHub repository. Whenever you make a change to the main branch of your repository, it'll deploy it to Neocities without any user input. A small note however, files you delete from the repository will have to be manually deleted from your Neocities site's files.

3. (Optional) GitHub Pages mirroring

There are a few extra steps needed to be able to mirror your Neocities site on GitHub Pages. First, make sure your repository is named [GitHub username].github.io. Then, make sure your site folder is called docs. This is because GitHub Pages can only be deployed from either the root folder or a /docs folder. You may also want to add a .gitattributes file to tell GitHub to not ignore code statistics on the /docs like it does by default. To do so, create a file called .gitattributes in your root directory and add the line docs/* linguist-documentation=false to it. Finally, set up deploying to GitHub Pages from your GitHub repository by going to Settings > Pages. Set the deploying branch to main and the deploying folder to /docs. Now every time you push to your repository, it'll automatically build and deploy your GitHub Pages mirror.
